# ACF field groups look broken in wp-admin (recovery)

The Next.js repo only **ships JSON**; it does **not** change your live WordPress database until you **deploy the theme** and **sync or import** field groups. If the editor looks “empty” or wrong, the **database** for field groups no longer matches the files on disk.

## 1. Confirm the theme on the server

You should have **all** of these under the active theme (path may vary):

- `acf-json/group_omb_*.json` — **10 files** (one per field group), each ending with a **`modified`** unix timestamp (regenerated by `npm run acf:extract-local-json`).
- **No** `acf-json/acf-import-bundle.json` (that bulk array file belongs only at `acf-import-bundle.json` in the **theme root** for `npm run acf:push`).
- Theme root `acf-import-bundle.json` — JSON **array** of 10 groups for REST import.

## 2. Force WordPress to reload definitions

1. Deploy/sync the theme folder so the server matches this repo.
2. Delete **`acf-json/acf-import-bundle.json`** on the server if it was copied there by mistake.
3. Log into wp-admin → **Custom Fields**.
4. Open the **Sync available** tab (ACF shows this when JSON differs from the DB). Select **all** OMB groups and sync.

If you **do not** see Sync:

- Run **`npm run acf:extract-local-json`** locally again (bumps `modified` on every group file), redeploy **`acf-json/`**, hard-refresh wp-admin, then check **Sync** again.

## 3. REST import (full overwrite from repo)

From your machine (with `.env.local`):

```bash
npm run acf:push
```

This POSTs `acf-import-bundle.json` to `/omb-headless/v1/acf-sync` and runs `acf_import_field_group` for each group. Requires the sync secret configured on WordPress.

## 4. Still broken — database repair

If definitions stay wrong after sync + push:

- Restore **`wp_posts` / `wp_postmeta`** for the **`acf-field-group`** post type from a **backup taken before the breakage**, or
- Re-import field groups from a known-good JSON backup using **ACF Tools → Import** (Pro), or
- Roll back the theme folder + DB together from hosting backups.

The frontend repo cannot reconstruct your production DB.
